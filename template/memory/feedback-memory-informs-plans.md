Before creating a plan, check corrections.md and any domain-relevant memories. Plans built on assumptions that corrections have already flagged as wrong will fail structurally — the plan encodes the error, the Generator implements it faithfully, and the Auditor must catch it in review (if at all).

**Why:** The M→S edge (Memory catalyzing Structure) is the weakest link in early governance. Without it, the planning process operates on the AI's training priors rather than learned context. This is how "first-session quality" plans happen in session 10 — the system has corrections but doesn't consult them before planning.

**How to apply:** At plan creation time (before source survey or audience analysis), scan corrections for anything relevant to the domain. If a correction touches the planned work, reference it explicitly in the plan's constraints section. For multi-phase plans, check memories that might affect sequencing or dependencies.
