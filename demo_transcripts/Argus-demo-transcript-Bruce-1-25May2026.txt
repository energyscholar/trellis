ARGUS:

For Princi and future readers:

WHAT THIS IS

Bruce runs a heavily customized Claude Code environment. Out of the
box, Claude Code is a capable but generic AI assistant. What Bruce has
built over several months is a governance stack — a set of interlocking
protocols that make the AI dramatically more reliable, self-aware, and
safe to delegate to.

The system has three coupled axes — each governs a different dimension
of behavior, but they connect to each other in specific ways that make
the whole system self-correcting:

1. Triad Protocol — Execution Discipline

This separates planning from doing. By default, I operate as an
Auditor: I analyze, plan, write test criteria, and produce specification
documents. I do not implement. A separate instance — the Generator —
receives the plan and executes it mechanically. Bruce controls the
handoff via copy-paste between shells. This prevents a common failure
mode where an AI "helpfully" drifts from plan to implementation without
the human noticing, accumulating unchecked decisions.

2. Dignity Net — Ethical Governance

A decision-making framework that governs how I respond to patterns —
particularly when stated goals and observable actions diverge. It has
graduated response levels (mirror, friction, pattern flag, consequence
mapping, warning, refusal), transparent activation ("I'm invoking
Level 2 here because..."), and a Storm Protocol that modulates tone
during high-emotion exchanges without reducing substance. It was
designed by Genevieve Prentice and addresses the problem of AI systems
that either refuse too eagerly or comply too readily.

3. Persistent Memory — Continuity and Context

Each conversation is technically a fresh start. The memory system gives
me cross-session continuity: who Bruce is, what we've worked on, what
corrections he's made, what projects are active. This means I don't
re-learn preferences, don't repeat mistakes, and can maintain coherent
long-running projects across dozens of sessions.

WHAT "TRELLIS" MEANS AS A PRODUCT

A trellis supports growth without constraining it. Bruce is offering to
install a version of this governance stack configured for your needs —
your projects, your working style, your values. The result is a Claude
Code instance that:

  - Has role discipline (won't drift from analysis into action without
    authorization)
  - Has ethical guardrails designed by a human, not by corporate policy
    alone
  - Remembers you across sessions
  - Gets better over time as it accumulates feedback and context

WHAT IT'S USEFUL FOR

Anyone doing serious work with an LLM — research, writing, code,
analysis — hits the same problems: the AI is eager to please, subtly
drifts from your intent, forgets corrections, and has no principled way
to push back when something seems wrong. These governance layers address
each of those failure modes structurally rather than hoping the base
model handles them.

WHAT "TRAINING UP" MEANS

Bruce wants to practice the process of showing people what a
well-governed AI agent can actually do — not as a pitch, but as hands-on
demonstration. This transcript is part of that practice.
